Workflow

Start with work, not settings.

Elph is intentionally small on the surface. It gives each coding session enough structure to move fast, while keeping the important decisions visible.

01

Open a repository

Run Elph where the code lives so it can load nearby instructions, current files, and project context.

02

Describe the outcome

Ask for the end state. Elph can inspect, search, edit, track todos, and keep changes scoped.

03

Approve and verify

Review tool actions, apply the change, and run focused tests before broader validation.

Capabilities

Minimal does not mean fixed.

The interface stays quiet, but the runtime is built to grow: native tools, model providers, prompts, and session controls are separated behind clear boundaries.

01

Focused TUI

Streaming responses, markdown rendering, keyboard-first controls, paste handling, and session logs.

02

Native tool loop

Read, Write, Edit, Grep, Glob, Bash, WebSearch, TodoList, skills, and media reading.

03

Provider routing

OpenAI-compatible APIs, OpenRouter routing, and Anthropic messages through layered adapters.

04

Approval workflow

Risky actions are gated, with session-level allow paths when a task needs more autonomy.

05

Memory-conscious runtime

History compaction, capped tool output, and deferred heavy work keep long sessions responsive.

Extend

Grow capability behind stable boundaries.

Elph keeps the daily experience simple by putting extension points where maintainers expect them: provider adapters, tool catalog entries, prompt templates, and project-level instructions.
